# Knowledge Base

> MCP-generated factual reference documents for the Bedrock Knowledge Base pipeline.

## Purpose

This folder contains domain-organised documents generated by MCP tools (Antigravity + MCP servers).
These files are synced to S3 → indexed by Bedrock Knowledge Base → embedded in Pinecone.

The Bedrock Publisher Lambda queries Pinecone for relevant chunks when generating polished `.mdx` articles.
The AgentBedrock chatbot queries the same KB to answer questions about project architecture and implementation.

## Pipeline

```
1. MCP tools scan cdk-monitoring repo → generate .md files here
2. CI (deploy-bedrock.yml sync-kb-docs job) → uploads to S3
3. Bedrock Knowledge Base → indexes into Pinecone
4. Publisher Lambda → queries Pinecone → Sonnet writes polished .mdx
5. Chatbot Agent → queries Pinecone → Haiku answers questions
```

## Document Standards

Every document in this knowledge base **must** include:

### Required Structure

- **YAML Frontmatter** with `title`, `doc_type`, `domain`, `tags`, `related_docs`, `last_updated`, `author`, `status`
- **`.metadata.json` sidecar file** for Bedrock retrieval filtering
- **`## Summary` section** (2–3 sentences) for chunk-0 semantic density
- **`## Keywords` section** with comma-separated search terms

### Content Rules

- All files must be factual — no opinions, no hallucinated details
- Code snippets, file paths, and commands must come from real source files
- Each file should be self-contained (readable without other KB files)
- Use UK English throughout
- No empty placeholder sections (e.g., `*No prerequisites detected*`)
- No HTML comments or formatting guidance from generators

### Content Creation Checklist

- [ ] YAML frontmatter present with all required fields
- [ ] `## Summary` section (2–3 sentences) at document start
- [ ] `## Keywords` section with search terms
- [ ] `## Source Files` with valid repository paths
- [ ] `## Related Documents` with links to relevant KB docs
- [ ] No placeholder/empty sections
- [ ] UK English throughout
- [ ] `.metadata.json` sidecar file generated
