# Knowledge Base

> MCP-generated factual reference documents for the Bedrock Knowledge Base pipeline.

## Purpose

This folder contains domain-organised documents generated by MCP tools (Antigravity + MCP servers).
These files are synced to S3 → indexed by Bedrock Knowledge Base → embedded in Pinecone.

The Bedrock Publisher Lambda queries Pinecone for relevant chunks when generating polished `.mdx` articles.

## Folder Structure

| Subfolder | MCP Tool | Content |
|:----------|:---------|:--------|
| `adrs/` | `generate-adr` | Architecture Decision Records with evidence |
| `code/` | `analyse-portfolio` (scoped) | Code analysis by domain (networking, observability, security) |
| `runbooks/` | `generate-runbook` | Operational incident runbooks |
| `implementation/` | `generate-technical-doc` | Detailed implementation walkthroughs |
| `live-infra/` | `mcp-infra-server` / `mcp-infra-diagram` | Live infrastructure reviews (when cluster reachable) |

## Pipeline

```
1. MCP tools scan cdk-monitoring repo → generate .md files here
2. CI (deploy-bedrock.yml sync-kb-docs job) → uploads to S3
3. Bedrock Knowledge Base → indexes into Pinecone
4. Publisher Lambda → queries Pinecone → Sonnet writes polished .mdx
5. Chatbot Agent → queries Pinecone → Haiku answers questions
```

## Rules

- All files must be factual — no opinions, no hallucinated details
- Code snippets, file paths, and commands must come from real source files
- Each file should be self-contained (readable without other KB files)
- Use UK English throughout
